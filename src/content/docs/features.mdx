## Ingestion engine

<CardGrid>
  <Card title="5.52M events/sec" icon="rocket">
    Lock-free MPMC ring buffer with Highway SIMD batch copy. Zero allocation on the hot path. Robotics telemetry, factory sensors, or tick feeds — same pipeline.
  </Card>
  <Card title="Protocol handlers" icon="star">
    Built-in FIX 4.4, ITCH, Binance WebSocket, and Kafka. OPC-UA, MQTT, and ROS2 on the roadmap. Plug custom feeds in via the C++ API.

    > Kafka connector requires an Enterprise license.
  </Card>
  <Card title="Durable WAL" icon="approve-check-circle">
    Write-ahead log for crash recovery. Configurable sync policy — every write, periodic, or async — per deployment profile.
  </Card>
</CardGrid>

## Query engine

<CardGrid>
  <Card title="LLVM JIT" icon="seti:clock">
    Queries compile to native code at runtime. Vectorized execution with SIMD aggregation across AVX2, AVX-512, and NEON.
  </Card>
  <Card title="ASOF JOIN" icon="star">
    Point-in-time correct joins across heterogeneous streams. Nanosecond precision. Foundational for sensor fusion, backtests, and multi-source telemetry.
  </Card>
  <Card title="Window JOIN" icon="star">
    Join within a time window (±N ms). Built for sensor clock drift, event correlation, and trade/quote alignment.
  </Card>
  <Card title="Temporal analytics" icon="rocket">
    EMA, VWAP, xbar, mavg, percentile — analytic primitives compiled in, not bolted on via UDFs.
  </Card>
  <Card title="Standard SQL" icon="approve-check-circle">
    Full SQL: SELECT, JOIN, GROUP BY, window functions, subqueries, CTEs. Your team already knows how to write it.
  </Card>
</CardGrid>

## Storage

| Feature | Description |
|---------|-------------|
| In-memory column store | Columnar layout optimized for SIMD scan and aggregation |
| Arena allocator | Custom memory management — no malloc on the hot path |
| Partition manager | Symbol-based partitioning for data locality and parallelism |
| Parquet HDB | Historical database on S3 / GCS / NFS. Hot + cold in one query |
| Compression | LZ4 for WAL, Parquet columnar compression for HDB |

## Security

- TLS 1.3 with optional mTLS on every endpoint, including cluster RPC
- RBAC with 5 built-in roles (admin, writer, reader, analyst, monitor)
- JWT / OIDC authentication (Okta, Auth0, Azure AD, Google Workspace)
- API key management with rotation and revocation
- Audit logging — all auth events, queries, and admin actions
- Rate limiting per user and per role
- Multi-tenancy with namespace isolation
- SOC2, MiFID II, and industrial (IEC 62443) evidence ready

> **Edition note:** TLS, basic RBAC, API keys, and rate limiting are in all editions. SSO/OIDC, audit export, and per-tenant advanced RBAC require Enterprise.

See the [Security Operations Guide](/api/security_operations_guide/) for configuration.

## Clustering

- Multi-node with automatic sharding by symbol
- Consistent hashing for partition placement
- Health monitoring and automatic failover
- Rolling upgrades with zero downtime
- Cross-node query routing and aggregation

> Multi-node clustering requires an Enterprise license.

See [Multi-Node Cluster](/deployment/multi_node_cluster/) for setup.

## Deployment

- Docker — single-command start for dev, CI, and edge

- Bare metal with NUMA / CPU pinning optimization

- Parquet HDB on S3, GCS, or local NFS
