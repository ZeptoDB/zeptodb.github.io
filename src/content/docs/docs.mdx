## Current Agent Memory operating model

Agent Memory supports routed writes, point reads, fan-out memory search/context assembly, semantic-cache fan-out, replica WAL durability policy, delete and eviction tombstones, tenant quotas, local stats, cluster-scoped stats, and owner-failover reporting.

Shard migration dual-write/catch-up remains future work. For implementation details, use [HTTP API Reference](/api/http_reference/#ai-agent-memory-api), [Python API Reference](/api/python_reference/#agent-memory-and-cache), and [Agent Memory for AI Agents](/use-cases/agent-memory/).
