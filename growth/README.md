# ZeptoDB Growth System

This folder is the operating base for the first week of ZeptoDB community-led growth.
The goal is not to build a spam bot. The goal is to build a repeatable system that finds
high-intent conversations, drafts useful responses, routes risky actions through review,
and measures which communities and messages create real product demand.

## Week 1 Scope

- Define the initial audience map and buying/use-case triggers.
- Seed the first community watchlist for Reddit, Threads, Hacker News, GitHub, Discord,
  Slack, LinkedIn, and specialist forums.
- Seed the first keyword bank for listening, content planning, and landing page routing.
- Establish a compliance gate for Reddit and Threads.
- Establish a UTM taxonomy so every public link is measurable.
- Run the first manual listening and approval loop before any automation posts publicly.

## Core Loop

1. Listen for relevant conversations.
2. Score the conversation by product fit, urgency, audience quality, and reply risk.
3. Draft a helpful response using ZeptoDB docs, benchmarks, and examples.
4. Review against community rules and platform policies.
5. Publish only when the response is useful without the link.
6. Track the link, reply outcome, docs visit, GitHub click, and contact/demo intent.
7. Feed outcomes back into the keyword bank and community watchlist.

## Week 1 Success Criteria

- 50 or more candidate communities are mapped.
- 100 or more listening keywords are seeded.
- Every external campaign link uses the UTM taxonomy.
- Reddit remains human-approved only.
- Threads publishing is limited to helpful, scheduled brand posts.
- At least 25 high-intent opportunities are logged by the end of the week.
- At least 5 responses or posts are published after review.

## Files

- `week-01-checklist.md`: Daily execution checklist for the first week.
- `audience-map.csv`: Initial ICP and use-case segments.
- `community-watchlist.csv`: Candidate communities and posting posture.
- `keyword-bank.csv`: Listening keywords, intents, and recommended landing pages.
- `compliance-checklist.md`: Platform rules and review gate.
- `utm-taxonomy.md`: Source, medium, campaign, and content naming rules.
- `content-pillars.md`: Message pillars and first-week post angles.

## Default Posture

When in doubt, use the system for discovery and drafting, not autonomous publishing.
Reddit especially should be treated as a trust channel: answer real questions, disclose
affiliation, and avoid repetitive links. Threads can carry more scheduled brand content,
but still needs pacing, reply monitoring, and proof-backed claims.
